What Is Machine Learning?

Understanding AI from first principles

Machine learning (ML) is a way of teaching computers to learn from data, instead of writing rules manually. Instead of saying "if cholesterol > 240 then risk is high", you feed the computer thousands of patient records and it figures out the rules itself.

💡

Simple Analogy

Imagine showing a child 1,000 photos of cats and dogs and saying "cat" or "dog" for each. After enough examples, the child learns what makes a cat a cat — without you ever explaining whiskers or ears explicitly. ML works the same way.

1.1 Types of Machine Learning

🏷️

Supervised Learning

Training on labeled data — each example has an input AND a known correct answer. This is what we use.

🔍

Unsupervised Learning

Finding patterns in unlabeled data. No correct answers given — the model groups similar things together.

🎮

Reinforcement Learning

Learning by trial and error with rewards and penalties. Used in game-playing AI like AlphaGo.

🔁

Semi-Supervised

A mix of labeled and unlabeled data. Useful when labeling data is expensive or time-consuming.

1.2 Supervised Learning in Detail

In supervised learning, we have a training dataset — rows of data where each row is one patient with known features (age, cholesterol, blood pressure…) and a known label (has heart disease: yes or no).

Our project is a binary classification problem — the model outputs either 0 (no disease) or 1 (has disease), along with a confidence percentage.

1.4 Model Evaluation

How do we know if our model is good? We split data into:

  • Training set (80%) — The model learns from this
  • Test set (20%) — Never shown to the model during training; used to check real-world performance
🎯

Accuracy

Percentage of correct predictions out of all predictions. Our ANN achieved 67.2%.

📈

ROC-AUC Score

Measures how well the model separates classes. Ranges 0.5–1.0. Our ANN: 0.86 (excellent).

🎗️

Precision

Of all patients predicted as "sick", how many actually were? Avoids false alarms.

🔎

Recall

Of all actually sick patients, how many did we catch? Critical for medical AI — missing sick people is dangerous.

The Three ML Models in This Project

Logistic Regression · Random Forest · Artificial Neural Network

2.1 Logistic Regression

Despite the name having "regression" in it, Logistic Regression is a classification algorithm. It calculates a probability (0 to 1) that a patient has heart disease.

P(disease) = 1 / (1 + e^−(w₁x₁ + w₂x₂ + ... + wₙxₙ + b))

Each feature (age, cholesterol, etc.) gets a weight (w). A high positive weight means that feature strongly predicts disease. The sigmoid function squishes the result into a probability between 0 and 1.

✅

Why use it?

Fast to train, easy to interpret, works well as a baseline. Doctors can understand "age has a weight of 0.3" — it's explainable.

2.2 Random Forest

Random Forest is an ensemble method — it builds many decision trees and combines their votes. Each tree is trained on a random subset of the data and features.

How a neuron works

output = activation( w₁·x₁ + w₂·x₂ + ... + b )

Each neuron takes inputs, multiplies them by weights, adds a bias, then passes through an activation function (like ReLU: max(0, x)) to introduce non-linearity.

Training process (Backpropagation)

  1. Forward pass — Input flows through the network, producing an output
  2. Calculate loss — Compare output to true label using Binary Cross-Entropy
  3. Backward pass — Calculate how much each weight contributed to the error
  4. Update weights — Adjust weights slightly using the Adam optimizer
  5. Repeat — For 100 epochs (full passes through the data)
🧠

Why use an ANN?

ANNs can learn complex non-linear relationships in data that simpler models miss. They are the foundation of modern deep learning (ChatGPT, image recognition, etc.).

3.3 Data Preprocessing Steps

Raw data must be cleaned and prepared before feeding to ML models:

🗑️

1. Remove Missing Values

Drop rows where any feature is missing. The UCI dataset uses ? for missing values — pandas drops these.

⚖️

2. Feature Scaling

Age (20–80) and cholesterol (100–400) are on different scales. StandardScaler normalizes them to mean=0, std=1.

✂️

3. Train/Test Split

80% training, 20% testing. random_state=42 ensures reproducible splits every run.

🔢

4. Binary Target

Convert multi-class target (0–4) to binary (0 or 1) for classification.

The actual preprocessing code:

import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

# Load the UCI dataset
df = pd.read_csv("heart_disease.csv")

# Remove missing values (marked as '?')
df = df.replace('?', pd.NA).dropna()

# Convert target to binary: 0 = no disease, 1 = disease
df['target'] = (df['target'] > 0).astype(int)

# Split features and label
X = df.drop('target', axis=1)
y = df['target']

# Normalize features (very important for ANN!)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 80/20 split
X_train, X_test, y_train, y_test = train_test_split(
    X_scaled, y, test_size=0.2, random_state=42
)
⚠️

Why is scaling important for ANN but less for Random Forest?

ANN uses gradient descent — large feature values create large gradients and slow/unstable training. Random Forest uses decision trees that split on thresholds, so scale doesn't matter.

4.3 Training the ANN with TensorFlow

import tensorflow as tf
from tensorflow.keras import layers, models

# Define the network architecture
model = models.Sequential([
    layers.Dense(64, activation='relu', input_shape=(13,)),  # 64 neurons, ReLU
    layers.Dropout(0.3),                                    # Prevents overfitting
    layers.Dense(32, activation='relu'),                    # 32 neurons
    layers.Dropout(0.2),
    layers.Dense(1, activation='sigmoid')                   # Output: probability
])

# Compile: how to measure error and how to fix it
model.compile(
    optimizer='adam',                    # Smart gradient descent
    loss='binary_crossentropy',           # Loss function for binary classification
    metrics=['accuracy']
)

# Train for 100 full passes through the data
history = model.fit(
    X_train, y_train,
    epochs=100,
    batch_size=32,
    validation_split=0.1,
    verbose=1
)

5.2 Request Flow: Making a Prediction

What happens step by step when a doctor clicks "Get Prediction":

  1. Doctor fills form on /predict page (age, cholesterol, etc.)
  2. React calls predictionsApi.predict(data) in app/lib/api.js
  3. HTTP POST request sent to http://localhost:8000/api/predictions/predict with JWT token in header
  4. FastAPI route in routes/predictions.py receives the request, verifies JWT
  5. Controller in controllers/prediction_controller.py loads the ML model from disk
  6. Scaler normalizes the 13 input features using the saved StandardScaler
  7. ML model predicts — returns probability (e.g., 0.73)
  8. Result saved to MySQL predictions table
  9. JSON response sent back: {"prediction": 1, "confidence": 73.0, "risk_level": "High"}
  10. React navigates to /results/[id] and displays charts and recommendations

FastAPI is a modern Python web framework for building APIs. It automatically generates documentation, validates data, and is extremely fast. It runs at http://localhost:8000.

6.2 Routes vs Controllers Pattern

The backend uses a clean separation of concerns:

FileResponsibilityExample

controllers/Business logic — load models, query database, compute resultsdef run_prediction(db, data)
# routes/predictions.py — THIN layer, just HTTP
from fastapi import APIRouter, Depends
from controllers.prediction_controller import run_prediction

router = APIRouter(prefix="/api/predictions")

@router.post("/predict")
def predict(data: PredictionInput, user=Depends(get_current_user), db=Depends(get_db)):
    return run_prediction(db, data, user.id)  # Delegate to controller


# controllers/prediction_controller.py — BUSINESS LOGIC
def run_prediction(db, data, user_id):
    scaler = joblib.load('models/scaler.pkl')
    model  = joblib.load('models/random_forest.pkl')

    features = [[data.age, data.sex, data.cp, ...]]
    scaled   = scaler.transform(features)
    prob     = model.predict_proba(scaled)[0][1]

    # Save to database
    prediction = Prediction(user_id=user_id, confidence=prob*100, ...)
    db.add(prediction); db.commit()

    return {"prediction": int(prob > 0.5), "confidence": prob*100}

# Creating a token
from jose import jwt

def create_access_token(data: dict):
    expire = datetime.utcnow() + timedelta(days=30)
    to_encode = {**data, "exp": expire}
    return jwt.encode(to_encode, "SECRET_KEY", algorithm="HS256")

# Verifying a token (runs on every protected request)
def get_current_user(token: str = Depends(oauth2_scheme)):
    payload = jwt.decode(token, "SECRET_KEY", algorithms=["HS256"])
    user_id = payload.get("sub")
    return db.query(User).filter(User.id == user_id).first()

6.4 Password Hashing

Passwords are never stored as plain text. bcrypt turns "mypassword" into "$2b$12$randomhash..." — irreversible, so even if the database is leaked, passwords are safe.

from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"])

# When registering
hashed = pwd_context.hash("mypassword")   # → "$2b$12$..."

# When logging in
pwd_context.verify("mypassword", hashed)   # → True or False

7.3 The API Client (app/lib/api.js)

All communication with the backend goes through one file. This keeps the HTTP logic in one place:

const BASE = 'http://localhost:8000';

function getToken() {
  return localStorage.getItem('token');
}

// Generic fetch helper — adds auth header automatically
async function request(url, options = {}) {
  const token = getToken();
  const res = await fetch(BASE + url, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      ...(token ? { 'Authorization': `Bearer ${token}` } : {}),
      ...options.headers,
    },
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}

// Usage in a React component:
const result = await predictionsApi.predict({
  age: 54, sex: 1, cp: 0, trestbps: 130, ...
});
// → { prediction: 1, confidence: 73.0, risk_level: "High" }
